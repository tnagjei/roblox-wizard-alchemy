# 英中多语言完整实施计划

## 0. 结论

本项目的多语言不能继续采用“英文数据源 + 运行时拼接翻译”的方式。正确方向是：

```text
公共 UI：JSON
页面正文：TS
```

也就是：

```text
messages/en.json
messages/zh-tw.json

content/i18n/en/home.ts
content/i18n/en/codes.ts
content/i18n/zh-tw/home.ts
content/i18n/zh-tw/codes.ts
```

核心目标：

```text
英文页面和繁中页面结构一致
模块一致
信息量一致
SEO 数据一致
只是用户可见文本变成对应语言
```

短期只做英文和繁中，不扩其他语言。等英文与繁中结构稳定，再复制到泰语、印尼语、越南语等语言。

---

## 1. 当前问题复盘

### 1.1 当前多语言实现的问题

当前代码里多语言逻辑分散在多个地方：

```text
lib/i18n-all.ts
lib/home-i18n.ts
components/LocalizedGuidePage.tsx
components/HomePageTemplate.tsx
app/[locale]/page.tsx
app/[locale]/[...slug]/page.tsx
app/zh-tw/page.tsx
app/zh-tw/codes/page.tsx
components/SiteChrome.tsx
```

主要问题：

```text
1. 翻译内容、页面模板、路由 fallback、SEO 逻辑混在一起。
2. 一些页面走独立静态页，一些页面走动态模板。
3. 动态模板仍会从英文 siteData.pages 自动生成内容。
4. 页面会出现中文标题 + 英文正文的混合情况。
5. 扩展其他语言时，会把当前混乱复制到更多语言。
```

### 1.2 当前不能继续做的方式

不要继续在这些文件里新增页面正文翻译：

```text
lib/i18n-all.ts
lib/home-i18n.ts
components/LocalizedGuidePage.tsx
components/HomePageTemplate.tsx
app/[locale]/[...slug]/page.tsx
```

这些文件可以短期保留以避免旧页面报错，但不应该再作为新多语言内容的主要承载层。

### 2.1 总体结构

```text

  en.json

content/
  i18n/
    en/
      home.ts
      codes.ts
      tier-list.ts
      updates.ts
      beginners-guide.ts
      units.ts
    zh-tw/
      home.ts
      codes.ts
      tier-list.ts
      updates.ts
      beginners-guide.ts
      units.ts

components/
  templates/
    HomePageTemplate.tsx
    CodesPageTemplate.tsx
    TierListPageTemplate.tsx
    UpdatesPageTemplate.tsx
    BeginnerGuideTemplate.tsx
    UnitsPageTemplate.tsx

lib/
  i18n/
    locales.ts
    routes.ts
    metadata.ts
  content/
    page-types.ts
```

### 2.2 四层职责

| 层级 | 目录 | 职责 | 不应该做什么 |
|---|---|---|---|
| 公共 UI 文案 | `messages/*.json` | 导航、按钮、footer、表格表头、状态标签 | 不放长篇 SEO 正文 |
| 页面正文 | `content/i18n/{locale}/{page}.ts` | 首页、Codes、Tier List 等完整正文 | 不写 React 组件 |
| 页面模板 | `components/templates/*.tsx` | 只负责展示内容 | 不硬编码翻译正文 |
| 工具层 | `lib/i18n/*`、`lib/content/*` | 语言列表、路径、metadata、类型 | 不放页面正文 |

## 5. 类型设计

### 5.1 `lib/content/page-types.ts`

建议先建立这些类型：

```ts
export type SeoMeta = {
  title: string;
  description: string;
};

export type HeroBlock = {
  eyebrow: string;
  h1: string;
  lede: string;
};

export type TextSection = {
  heading: string;
  body: string[];
  list?: string[];
};

export type FaqItem = {
  q: string;
  a: string;
};

export type HomePageContent = {
  meta: SeoMeta;
  hero: HeroBlock;
  directory: {
    eyebrow: string;
    title: string;
    cards: Array<{
      href: string;
      title: string;
      description: string;
    }>;
  };
  research: {
    eyebrow: string;
    title: string;
    cards: Array<{
      title: string;
      description: string;
    }>;
  };
  popularSearches: Array<{
    href: string;
    title: string;
    description: string;
    covers: string;
  }>;
  faq: FaqItem[];
};

export type CodesPageContent = {
  meta: SeoMeta;
  hero: HeroBlock;
  verifiedCodes: {
    eyebrow: string;
    title: string;
    description: string;
  };
  communityCodes: {
    eyebrow: string;
    title: string;
    description: string;
  };
  pendingCodes: {
    eyebrow: string;
    title: string;
    description: string;
  };
  conflictMatrix: {
    eyebrow: string;
    title: string;
    description: string;
  };
  redeemSteps: string[];
  sections: TextSection[];
  faq: FaqItem[];
};
```

### 5.2 为什么一定要类型

原因：

```text
1. 防止少字段。
2. 防止数组结构不一致。
3. 防止 Vercel build 才发现 page.tsx 类型错误。
4. 方便未来复制到其他语言。
```

## 6. 页面模板设计

### 6.1 模板目录

```text
components/templates/HomePageTemplate.tsx
components/templates/CodesPageTemplate.tsx
components/templates/TierListPageTemplate.tsx
components/templates/UpdatesPageTemplate.tsx
components/templates/BeginnerGuideTemplate.tsx
components/templates/UnitsPageTemplate.tsx
```

### 6.2 模板原则

```text
1. 模板只负责渲染 content。
2. 不在模板里写繁中或英文长正文。
3. 不在模板里判断 locale 后输出不同正文。
4. 允许模板渲染动态数据，例如 siteData.codes.communityReportedCodes。
5. 动态数据的表头和说明必须来自 content 或 messages。
```

### 6.3 Codes 模板示例

```tsx
import type { CodesPageContent } from "@/lib/content/page-types";

export function CodesPageTemplate({ content }: { content: CodesPageContent }) {
  return (
    <main className="page-main">
      <section className="page-hero compact-hero">
        <div className="hero-copy">
          <p className="eyebrow">{content.hero.eyebrow}</p>
          <h1>{content.hero.h1}</h1>
          <p className="lede">{content.hero.lede}</p>
        </div>
      </section>

      {content.sections.map((section) => (
        <article className="guide-card" key={section.heading}>
          <h2>{section.heading}</h2>
          {section.body.map((paragraph) => (
            <p key={paragraph}>{paragraph}</p>
          ))}
          {section.list && section.list.length > 0 ? (
            <ul>
              {section.list.map((item) => (
                <li key={item}>{item}</li>
              ))}
            </ul>
          ) : null}
        </article>
      ))}
    </main>
  );
}
```

## 8. SEO 与路径

### 8.1 语言配置

```text
lib/i18n/locales.ts
```

```ts
export const locales = ["en", "zh-tw"] as const;

export type Locale = (typeof locales)[number];

export const localeMeta = {
  en: {
    label: "English",
    hreflang: "en"
  },
  "zh-tw": {
    label: "繁體中文",
    hreflang: "zh-TW"
  }
} satisfies Record<Locale, { label: string; hreflang: string }>;
```

### 8.2 路径工具

```text
lib/i18n/routes.ts
```

```ts
import type { Locale } from "./locales";

export function getLocalizedPath(locale: Locale, slug: string) {
  const cleanSlug = slug.replace(/^\/+|\/+$/g, "");

  if (locale === "en") {
    return cleanSlug ? `/${cleanSlug}/` : "/";
  }

  return cleanSlug ? `/${locale}/${cleanSlug}/` : `/${locale}/`;
}

export function getSlugFromPath(pathname: string) {
  return pathname
    .replace(/^\/(zh-tw|th|id|vi|ko|ru|es|pt-br)(?=\/|$)/, "")
    .replace(/^\/+|\/+$/g, "");
}
```

### 8.3 Metadata 工具

```text
lib/i18n/metadata.ts
```

```ts
import { absoluteUrl } from "@/lib/seo";
import type { Locale } from "./locales";
import { getLocalizedPath } from "./routes";

export function buildLocalizedMetadata({
  locale,
  slug,
  title,
  description
}: {
  locale: Locale;
  slug: string;
  title: string;
  description: string;
}) {
  return {
    title,
    description,
    alternates: {
      canonical: absoluteUrl(getLocalizedPath(locale, slug)),
      languages: {
        en: absoluteUrl(getLocalizedPath("en", slug)),
        "zh-TW": absoluteUrl(getLocalizedPath("zh-tw", slug)),
        "x-default": absoluteUrl(getLocalizedPath("en", slug))
      }
    }
  };
}
```

## 9. 执行顺序

### 阶段 1：建立标准骨架

```text
1. 新建 messages/en.json
2. 新建 messages/zh-tw.json
3. 新建 lib/content/page-types.ts
4. 新建 lib/i18n/locales.ts
5. 新建 lib/i18n/routes.ts
6. 新建 lib/i18n/metadata.ts
```

### 阶段 2：迁移首页

```text
1. 新建 content/i18n/en/home.ts
2. 新建 content/i18n/zh-tw/home.ts
3. 新建 components/templates/HomePageTemplate.tsx
4. 改 app/page.tsx 使用 enHomePage
5. 改 app/zh-tw/page.tsx 使用 zhTwHomePage
6. npm run build
7. 检查 / 和 /zh-tw/
```

### 阶段 3：迁移 Codes 页

```text
1. 新建 content/i18n/en/codes.ts
2. 新建 content/i18n/zh-tw/codes.ts
3. 新建 components/templates/CodesPageTemplate.tsx
4. 改 app/codes/page.tsx 使用 enCodesPage
5. 改 app/zh-tw/codes/page.tsx 使用 zhTwCodesPage
6. 移除 app/[locale]/[...slug]/page.tsx 中 zh-tw/codes 的特殊兜底
7. npm run build
8. 检查 /codes/ 和 /zh-tw/codes/
```

### 阶段 4：迁移繁中核心页

按顺序做：

```text
1. tier-list
2. updates
3. beginners-guide
4. units
```

每页都按同样模式：

```text
content/i18n/en/{page}.ts
content/i18n/zh-tw/{page}.ts
components/templates/{Page}Template.tsx
app/{page}/page.tsx
app/zh-tw/{page}/page.tsx
```

### 阶段 5：清理旧逻辑

当英文和繁中核心页都稳定后，再处理：

```text
1. 降低或删除 components/LocalizedGuidePage.tsx 的使用
2. 清理 lib/i18n-all.ts 中的正文自动拼接逻辑
3. 清理 lib/home-i18n.ts
4. 简化 app/[locale]/[...slug]/page.tsx
5. 保留 sitemap 和 hreflang 的路径生成能力
```

## 10. 翻译提示词

### 10.1 JSON 公共 UI 翻译提示词

```text
你是一个资深网站本地化工程师，正在为一个 Roblox 攻略站做 UI 文案多语言。

任务：
请根据我提供的 messages/en.json，生成 messages/zh-tw.json。

严格要求：
1. 保持 JSON 结构、key、层级、顺序完全一致。
2. 只能翻译 value，绝不能修改 key。
3. 输出必须是合法 JSON，不要加注释，不要加 Markdown，不要解释。
4. 使用繁體中文，不要使用简体中文。
5. 语言要自然，适合台湾、香港和海外繁中用户阅读。
6. 保留以下专有名词，不要翻译：
   Noob Tower Defense
   Roblox
   Vitan Games
   Codes
   Tier List
   Units
   Value List
   Discord
   YouTube
   Macro
   Loadout
   Gems

8. 如果英文是技术名词或产品名，优先保留英文。
9. 如果 value 是 URL、路径、email、代码、变量占位符，不要翻译。
10. 保留所有占位符，例如 {count}、{date}、{code}、%s、{{name}}。

翻译风格：
不要像机器翻译。
不要过度书面。
不要大陆化表达。
```

## 11. 扩展到其他语言

### 11.1 扩展前提

只有在以下条件满足后，才扩其他语言：

```text
1. 英文首页和 Codes 页已经迁移到 content/i18n/en。
2. 繁中首页和 Codes 页已经迁移到 content/i18n/zh-tw。
3. HomePageTemplate 和 CodesPageTemplate 稳定。
4. npm run build 通过。
5. / 和 /zh-tw/ 页面人工检查通过。
6. /codes/ 和 /zh-tw/codes/ 页面人工检查通过。
```

### 11.2 扩展语言顺序

建议顺序：

```text
1. th
2. id
3. vi
4. ko
5. ru
6. es
7. pt-br
```

### 11.3 扩展单个语言的流程

以泰语为例：

```text
1. messages/en.json → messages/th.json
2. content/i18n/en/home.ts → content/i18n/th/home.ts
3. content/i18n/en/codes.ts → content/i18n/th/codes.ts
4. 在 lib/i18n/locales.ts 注册 th
5. 添加 /th/ 和 /th/codes/ 路由
6. npm run build
7. 人工检查 /th/ 和 /th/codes/
8. 上线后观察 GSC 展示和点击
```

### 11.4 扩展其他语言的 JSON 提示词

```text
请基于 messages/en.json 生成 messages/{locale}.json。

目标语言：{目标语言}
目标语言代码：{locale}

要求：
1. 保持 JSON key、层级、顺序完全一致。
2. 只翻译 value。
3. 保留 Noob Tower Defense、Roblox、Vitan Games、Codes、Tier List、Units、Value List、Discord、YouTube、Macro、Loadout、Gems。
4. URL、路径、变量占位符不要翻译。
5. 文案要适合当地玩家，不要机器翻译。
6. 输出合法 JSON，不要解释。

输入：
[粘贴 messages/en.json]
```

### 11.5 扩展其他语言的 TS 提示词

```text
请基于 content/i18n/en/{page}.ts 生成 content/i18n/{locale}/{page}.ts。

目标语言：{目标语言}
目标语言代码：{locale}

要求：
1. 保持 TS 文件结构、字段名、数组顺序完全一致。
2. 只翻译用户可见文本。
3. 不翻译变量名、字段名、import、URL、slug。
4. 保留 Noob Tower Defense、Roblox、Vitan Games、Codes、Tier List、Units、Value List、Discord、YouTube、Macro、Loadout、Gems。
5. SEO title 和 description 要适合当地搜索用户。
6. FAQ、表格表头、步骤、CTA、正文必须完整本地化。
7. 不要新增事实，不要把未验证内容写成已验证。
8. 输出完整 TS 文件，不要 Markdown，不要解释。

输入：
[粘贴 content/i18n/en/{page}.ts]
```

每完成一个页面，必须检查：

```text

2. meta title 是否本地化。
3. meta description 是否本地化。
4. 首屏 lede 是否本地化。

6. 所有正文段落是否本地化。
7. 所有表格表头是否本地化。
8. 所有 CTA 是否本地化。
9. FAQ 是否本地化。
10. 内链是否指向当前语言路径。
11. canonical 是否指向当前语言页面。
12. hreflang 是否包含 en 和 zh-TW。
```

```text
YouTube

Loadout
```

## 13. 风险控制

### 13.1 不要一次性重构全站

风险：

```text
1. 容易引入 build 错误。
2. 容易让已有页面失效。
3. 不容易定位问题。
```

建议：

```text
先迁移 home
再迁移 codes
再迁移 tier-list
再迁移 updates
```

### 13.2 不要同时扩 8 种语言

风险：

```text
1. 翻译质量不可控。
2. 检查成本过高。
3. 错误会被复制到所有语言。
```

建议：

```text
先 en + zh-tw
稳定后再 th
再 id
再 vi
```

### 13.3 不要把 SEO 正文塞进 JSON

原因：

```text
1. 长正文难维护。
2. JSON 不能写注释。
3. 字段缺失不容易发现。
4. 页面结构复杂时阅读体验差。
```
